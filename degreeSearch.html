Degree Search

Degree Search interface for graduate students

My Contribution

Role
I was the Lead Designer in this project. My job was to insure that the application had consistence across both undergraduate and graduate degrees.
Project Type
Web Application
Project Duration
June 2023 to March 2024
Design Tool
Figma, Adobe Photoshop, Indesign, University Design System

University of Arizona

University of Arizona is a state university located in Tucson Arizona.

Overview

The Graduate Degree Search interface is designed to help users explore and filter graduate programs (e.g., Master’s, PhD) based on criteria such as discipline, location, institution, and program details. The goal is to provide an intuitive, efficient, and visually appealing platform that simplifies the complex process of researching higher education opportunities. This analysis evaluates the interface’s usability, aesthetics, functionality, and user experience, reflecting on the design process and outcomes.

Design Objectives:

  • User-Centric Navigation: Enable users to quickly find relevant programs with minimal friction.
  • Comprehensive Filtering: Offer robust filters to narrow down search results based on user preferences.
  • Responsive Design: Ensure accessibility and usability across devices (desktop, tablet, and mobile).
  • Clear Information Architecture: Present program details in a structured, digestible format.
  • Engaging Aesthetics: Use modern design principles that not only create a visually cohesive and professional interface, but also keep in mind the branding of the university.

UI/UX Analysis:

  • Usability
    The interface prioritizes ease of use through intuitive navigation and clear calls-to-action (CTAs). Key usability features include:
    • Search Functionality: A prominent search bar allows users to input keywords (e.g., “Computer Science Master’s”), with autocomplete suggestions to enhance efficiency.
    • Filter System: Filters for program type, location, tuition, and duration are logically organized, likely in a sidebar or dropdown menu, enabling users to refine results without overwhelming complexity.
    • Results Layout: Programs are displayed in a clean list or grid format, with key details (e.g., university name, degree type, duration) visible upfront, reducing cognitive load.
    • Accessibility: Assuming adherence to WCAG guidelines, the design likely incorporates sufficient color contrast, keyboard navigation, and screen-reader compatibility to ensure inclusivity.

    • Strengths:
      • Streamlined search process reduces time to find relevant programs.
      • Logical filter hierarchy supports both novice and experienced users.
      • Clear CTAs (e.g., “View Program” or “Apply Now”) guide user actions.

    • Areas for Improvement:
      • If filters are too numerous, consider progressive disclosure to avoid overwhelming users.
      • Ensure mobile users can easily access and manage filters without excessive scrolling.
      • Conduct usability testing to identify any friction points in the search-to-results flow.

  • Aesthetics
    The visual design likely adheres to modern UI trends, balancing functionality with an engaging aesthetic:
    • Typography: typography: Clean, readable fonts (e.g., sans-serif like Roboto or Inter) ensure legibility across program descriptions and headings.
    • Color Scheme: A professional palette (e.g., neutral tones with accent colors like blue or green) conveys trust and clarity, aligning with educational platforms.
    • Whitespace: Ample spacing between elements prevents visual clutter, enhancing focus on key content like search results.
    • Visual Hierarchy: Bold headings, consistent button styles, and highlighted CTAs guide user attention effectively.

    • Strengths:
      • Professional and approachable aesthetic aligns with the academic audience.
      • Consistent design language reinforces brand reliability.
      • Use of icons or visual cues (e.g., for filters or program types) enhances scannability.

    • Areas for Improvement:
      • Ensure color contrast meets accessibility standards (e.g., WCAG 2.1 AA).
      • Avoid overly decorative elements that could distract from core functionality.
      • Test visual consistency across devices to maintain a cohesive experience.

  • Functionality
    The interface is designed to handle complex data while maintaining performance and reliability:
    • Search and Filter Logic: The system likely supports dynamic filtering, updating results in real-time as users adjust criteria, with smooth transitions to maintain context.
    • Data Presentation: Program details are likely organized in cards or tables, with expandable sections for in-depth information (e.g., admission requirements, deadlines).
    • Responsive Design: The layout adapts seamlessly across devices, with mobile-friendly touch targets and collapsible menus for smaller screens.
    • Error Handling: Clear feedback for invalid inputs (e.g., “No results found”) guides users to refine their search.

    • Strengths:
      • Real-time filtering enhances interactivity and user control.
      • Responsive design ensures accessibility for diverse user contexts.
      • Structured data presentation supports quick decision-making.

    • Areas for Improvement:
      • Optimize backend performance for large datasets to prevent lag in results display.
      • Include a “reset filters” option for users to easily start over.
      • Test edge cases (e.g., no results or invalid queries) to ensure graceful handling.

  • User Experience
    The overall experience aims to empower users to make informed decisions about graduate programs:
    • User Flow: The journey from landing page to search to detailed program view is streamlined, with minimal steps to access critical information.
    • Feedback Mechanisms: Visual cues (e.g., loading spinners, highlighted filters) keep users informed of system status.
    • Personalization: If implemented, features like saved searches or favorite programs enhance user engagement by tailoring the experience.
    • Emotional Impact: The design likely conveys trust and clarity, reducing the stress of navigating complex academic options.

    • Strengths:
      • Intuitive flow aligns with user goals (finding and comparing programs).
      • Feedback mechanisms maintain transparency during interactions.
      • Clean design reduces decision fatigue for users exploring multiple options.

    • Areas for Improvement:
      • Consider adding onboarding tips or tooltips for first-time users unfamiliar with advanced filters.
      • Incorporate user feedback loops (e.g., surveys) to refine the experience.
      • Explore personalization features (e.g., recommended programs based on search history) to increase engagement.

Design Process:

The development of this interface likely followed a user-centered design process:

  • Research: Conducted user interviews or surveys to understand the needs of prospective graduate students (e.g., prioritizing program reputation, cost, or location).
  • Wireframing: Created low-fidelity wireframes to map out key components (search bar, filters, results) and user flows.
  • Prototyping: Built interactive prototypes in Figma to test navigation and filter interactions, iterating based on feedback.
  • Visual Design: Developed a style guide (typography, colors, components) to ensure consistency and alignment with educational branding.
  • Usability Testing: Conducted tests with target users to identify pain points, refining the design for clarity and efficiency.
  • Iteration: Incorporated feedback to enhance accessibility, responsiveness, and functionality.

Impact and Outcomes:

  • User Impact: The interface simplifies the graduate program search process, enabling users to make informed decisions efficiently.
  • Business Impact: For stakeholders (e.g., universities or platforms hosting the tool), the design likely increases user engagement and conversion rates (e.g., program applications).
  • Metrics of Success: Potential KPIs include time to find a program, filter usage rates, and user satisfaction scores from testing.

Reflections:

This project showcases a balance of functionality and aesthetics, addressing the complex needs of academic users while maintaining simplicity. Key lessons include the importance of iterative testing to refine filters and the value of responsive design for broad accessibility. Future iterations could explore advanced features like AI-driven program recommendations or integration with application portals to further enhance the user journey.
